Manual de Bases de Datos

Guía técnica para la asignatura de Implantación de Sistemas

1. Fases del Diseño

En la implantación de sistemas, la base de datos no se crea de la nada. Sigue un ciclo de vida estructurado para garantizar que los datos sean íntegros y el sistema eficiente.

Las 3 Etapas Críticas

  1. Diseño Conceptual: Es la etapa de abstracción. Se identifican las necesidades del negocio sin pensar en tecnología. El resultado es el diagrama E-R.
  2. Diseño Lógico: Se transforma el concepto en estructuras de datos (tablas). Aquí se aplican las reglas de normalización para evitar duplicidad.
  3. Diseño Físico: Es la implementación real en el servidor. Involucra configuración de almacenamiento, índices y seguridad.

2. Modelo Entidad-Relación (E-R)

Herramienta visual para representar la estructura lógica de la base de datos.

Elemento Descripción Representación
Entidad Objeto real o abstracto (Cliente, Factura). Rectángulo
Relación Asociación entre entidades (Compra, Pertenece). Rombo
Atributo Propiedad de la entidad (Nombre, Precio). Elipse / Lista
Nota de Implantación: Un error en el modelo E-R es costoso de corregir una vez que el sistema está en producción. Dedica tiempo a validar las relaciones.

3. Normalización

Proceso matemático para organizar los datos y minimizar la redundancia. En sistemas empresariales, buscamos llegar hasta la Tercera Forma Normal (3FN).

Reglas Esenciales

  • 1FN (Atomicidad): Cada celda de la tabla debe contener un único valor. Nada de listas separadas por comas.
  • 2FN (Dependencia Total): Todo atributo que no sea clave debe depender de toda la clave primaria (vital en claves compuestas).
  • 3FN (Independencia): No deben existir dependencias entre columnas que no son clave. Ej: La columna "NombreProvincia" no debe depender de "CodigoPostal" si la clave es "ID_Usuario".

4. Selección del SGBD

La decisión tecnológica más importante antes de instalar el software.

SQL (Relacionales)

Estructurados, rígidos y seguros. Ideales para sistemas transaccionales (ERP, CRM, Contabilidad).

  • MySQL / MariaDB: Estándar web, licencia flexible.
  • PostgreSQL: El más avanzado y robusto de código abierto.
  • SQL Server: Integración nativa con entornos corporativos Microsoft.

NoSQL (No Relacionales)

Flexibles y escalables. Ideales para Big Data, IoT o contenido dinámico.

  • MongoDB: Basado en documentos JSON.
  • Redis: Almacenamiento clave-valor en memoria (velocidad extrema).

5. Implantación Física

Pasos técnicos para poner el sistema en marcha (Deployment).

1. DDL (Data Definition Language)

Es el script que crea la estructura. Ejemplo de creación de una tabla optimizada:

CREATE TABLE Usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(150) UNIQUE NOT NULL,
    password_hash CHAR(64) NOT NULL,
    fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_email (email) -- Índice para búsquedas rápidas
) ENGINE=InnoDB;

2. Conexión desde el Sistema

Configuración típica en un archivo de entorno (.env) para seguridad:

DB_HOST=192.168.1.50
DB_PORT=3306
DB_NAME=sistema_ventas
DB_USER=app_user
DB_PASS=secret_secure_password
